Lý do mình dùng GitHub để viết blog

Lý do quan trọng nhất là mình không muốn tốn tiền. Chả là mình đang hăng say viết 1 chiếc blog siêu bé xinh xinh thì đùng, mình không thể gõ LateX trên WordPress (LaTeX là gì?). Sau một hồi mò mẫm thì mình cũng tìm ra được lý do. Đó là mình dùng WordPress free, mà free thì không thể cài plugin hỗ trợ gõ LaTeX. Muốn cài phải nâng cấp lên loại tài khoản gì gì đó mà tận 300$/1 năm, một con số không tưởng với một thằng sinh viên đang mốc mồm như mình :(

Vậy là mình bỏ WordPress từ đây :)))). Rồi mình chuyển sang sử dụng Github. Lý do cũng đơn giản là nó miễn phí, ngoài ra mình cũng có thể học thêm cách viết Markdown.

Hướng dẫn tạo Blog cá nhân sử dụng GitHub

Sau khi xác định phải viết Blog bằng GitHub thì mình bắt đầu mò mẫm, rồi cuối cùng cũng cho ra được bài đầu tiên - đó là bài này :). Đo thời gian mò mẫm cũng mất khá thời gian nên mình viết bài này với mong muốn tiết kiệm được chút thời gian cho ai đó muốn bắt đầu viết Blog trên GitHub như mình.

Nguyên liệu cần có

  1. Tài khoản GitHub
  2. Một chiếc máy tính
  3. Một chút thời gian

Các bước thực hiện

Tạo Repository

Sau khi đăng nhập GitHub, hãy tạo một Repository với định dạng: “username”.github.io và để chế độ Public. Ví dụ: khnhkdblog.github.io

Cài đặt Hugo

Hiểu đơn giản Hugo là công cụ giúp ta convert các file định dạng markdown sang các file html, css, js.

Hướng dẫn cài đặt Hugo từ trang chủ

Sau khi cài đặt, bật Terminal nhập lệnh: hugo version. Nếu màn hình trả về kết quả như dưới là đã cài đặt thành công.

Tạo nơi chứa Blog ở máy local

Mở Terminal và cd đến nơi mà ta muốn chứa Blog và nhập lệnh: hugo new site <site_name>

Chọn giao diện có sẵn

Sau khi tạo site, ta tiến hành chọn các giao diện mà Hugo hỗ trợ sẵn. Link

Ví dụ ở đây mình chọn Clean White

Nhấn Download sẽ hiện ra source code và hướng dẫn, ta đọc hướng dẫn rồi tiến hành clone về mục themes trong site vừa tạo ở bước trước.
Đoạn người ta hướng dẫn clone về máy

Chú ý: trước khi clone theme về máy thì hãy chạy lệnh: git init để khởi tạo git.

Cấu trúc site sau khi clone theme về máy

Cấu hình file config.toml

Ta mở tiến hành copy nội dung của file <site_name>/themes/<theme_name>/exampleSite/config.toml vào file <site_name>/config.toml.

Chạy server lần đầu

Sau khi đã có file config.toml, ta tiến hành chạy server lần đầu xem thành công hay thất bại nhé. Trong terminal, ta chạy lệnh: hugo server -D

Sau khi chạy server, ta vào link http://localhost:1313/ để kiểm tra, nếu màn hình hiện như hình dưới là đã chạy thành công.

Chỉnh sửa file config.toml

Tại vì file config.toml ta copy từ theme tải về sẵn, nên nó có khá đầy đủ các thành phần. Tuy nhiên nhu cầu sử dụng của chúng ta lại không cần thiết có nhiều thành phần như vậy. Vậy nên ta cần loại bỏ một số thành phần không cần thiết.

Ở bước này mình khuyên bạn là hãy tự chỉnh sửa file config.toml theo nhu cầu cá nhân. Ban đầu đọc file config.toml thì mình không hiểu, nhưng mình cứ chạy server, thử xoá một vài dòng trong file config.toml rồi lưu lại, xem sự thay đổi trên giao diện web để xác định đó là thành phần nào là hiểu ngay.

Bước này cũng khá tốn thời gian để hiểu rõ thành phần và loại bỏ thành phần dư thừa.
Kết quả sau khi dọn dẹp file config.toml
Và đây là thành phẩm

Chú ý:

  1. Ta lưu ảnh ở folder <site_name>/static/img
  2. Trong file config.toml, ta sửa baseurl = “https://<username>.github.io”

Tạo bài viết đầu tiên

Quay trở lại terminal, ta gõ lệnh: hugo new post/<blog_name>.md.

File Markdown vừa tạo sẽ được lưu trữ tại: <site_name>/content/post/<blog_name>.md

Viết gì đó vào file <blog_name>.md và lưu lại

Chạy lại server: hugo server -D và đây là kết quả:

Đẩy lên Repository GitHub

Trước tiên, ở thư mục <site_name>, ta tạo một thư mục có tên “username”.github.io (chính là tên Repository tạo ở bước đầu tiên)

Tiếp theo, trong file <site_name>/config.toml ta thêm dòng publishdir = “username.github.io”. Đây chính là folder ta dùng để chứa các file html, css, js sau khi convert từ file Markdown.

Chú ý: Nếu trong file viết blog (định dạng md) có chứa dòng draft: true thì đổi thành draft: false nhé, vì khi tạo bài viết bằng lệnh hugo new post/<blog_name>.md thì mặc định là bản nháp.

Tiếp theo ta vào thư mục “username”.github.io bằng lệnh cd. Khởi tạo git bằng lệnh git init, kết nối với Repository GitHub bằng lệnh git remote add origin https://github.com/<username>/<username>.github.io.git

Tiếp tục ta quay trở lại thư mục <site_name> bằng lệnh cd .., chạy lệnh hugo. Lệnh này sẽ convert các file định dạng Markdown về html, css, js và lưu ở folder “username”.github.io

Cuối cùng vào folder “username”.github.io trong terminal, chạy lệnh git add ., tiếp theo git commit -m “message”, cuối cùng git push –set-upstream origin master.

Chú ý ở lần push đầu tiên có thể bạn sẽ được yêu cầu nhập tài khoản và mật khẩu GitHub.

Thành quả

Cuối cùng là truy cập: https://github.com/<username>/<username>.github.io để tận hưởng thành quả nhé.
